The PEAS Template — Memorise This Structure

// For ANY agent question, fill this template: P — Performance Measure: ✓ Primary goal achieved? (delivery success, goal reached) ✓ Efficiency metric (battery saved, moves minimised, time) ✓ Safety penalty (collisions = -X points, blocked = -Y) ✓ Accuracy (correct room, correct item delivered) E — Environment: ✓ Physical space (hospital corridors / lunar surface / warehouse grid) ✓ Obstacles (blocked corridors / craters / walls) ✓ Other agents present (staff, pedestrians, other robots) ✓ Resources/constraints (battery, weight limits, time limits) A — Actuators: ✓ Movement: wheels/motors/thrusters ✓ Manipulation: robotic arm / gripper / dispenser ✓ Communication: antenna / display / speaker ✓ Task-specific: sample collector / medicine tray S — Sensors: ✓ Vision: camera / visual sensor ✓ Navigation: GPS / IMU / odometer / map ✓ Proximity: infrared sensor / sonar / lidar ✓ Task-specific: microphone / battery sensor / spectrometer

The Search Problem Template — 6 Components, Always in This Order

1. STATE: A complete description of everything the agent needs to know to determine what to do next. Example: (current_location, packages_remaining, battery_level) 2. START STATE: The initial configuration. Example: (pharmacy, all_medicines_undelivered, 100_units) 3. ACTIONS / SUCCESSOR FUNCTION: What moves are possible from any state. What new state each action produces. Example: Move to any non-blocked adjacent room → updates location, battery -= 1 4. GOAL TEST: The condition that defines "done". Example: current_location == target_room AND medicine delivered 5. PATH COST: How to measure the total cost of a solution. Example: Number of moves taken = battery units consumed 6. (Optional bonus) TOTAL STATES: If asked: |locations| × 2^|packages| × battery_levels

Q2 — BFS / DFS / UCS Graph Search

4–6 MARKS · MECHANICAL

This question is 100% mechanical. You trace through a graph using a queue/stack/priority-queue. The same graph (S→A→B→C→G style) appears every exam. Master the procedure and you'll never lose a mark here.

The Graph That Keeps Appearing (Assignment Q6 / Fall 2025 Q2)

// Edges (undirected unless stated otherwise): S ─── A (cost 1) S ─── C (cost 10) A ─── B (cost 5) A ─── C (cost 10) B ─── C (cost 2) B ─── G (cost 5) C ─── G (cost 1) // TIE-BREAKING: alphabetical (S before A before B...) // This graph or a close variant WILL appear in your exam.

BFS Procedure — Step by Step

Queue: [S] | Visited: {S} Pop S → expand: A(via S), C(via S) [alphabetical: A first] Queue: [A, C] | Visited: {S,A,C} Pop A → expand: B(not visited), C(visited→skip), S(visited→skip) Queue: [C, B] | Visited: {S,A,C,B} Pop C → expand: G(not visited), rest visited Queue: [B, G] | Visited: {S,A,C,B,G} Pop B → expand: G(already in queue!) // In graph search: if G already visited/in queue, skip! Pop G → GOAL FOUND! Trace back parents: G←C←S BFS Path: S→C→G | Cost: 10+1 = 11

UCS Procedure — Step by Step

PQ (priority queue, sorted by g): Push: (g=0, S) Pop (0, S). Mark S visited. Expand: A: g=0+1=1 → push (1, A, parent=S) C: g=0+10=10 → push (10, C, parent=S) PQ: [(1,A), (10,C)] Pop (1, A). Mark A visited. Expand: B: g=1+5=6 → push (6, B, parent=A) C: g=1+10=11 → push (11, C via A) [note: C also at g=10 via S] S: visited → skip PQ: [(6,B), (10,C via S), (11,C via A)] Pop (6, B). Mark B visited. Expand: C: g=6+2=8 → push (8, C, parent=B) G: g=6+5=11 → push (11, G via B) A: visited → skip PQ: [(8,C via B), (10,C via S), (11,C via A), (11,G via B)] Pop (8, C via B). Mark C visited. Expand: G: g=8+1=9 → push (9, G, parent=C via B) S,A,B: visited → skip PQ: [(9,G via C), (10,C via S — skip, C visited), (11,...)] Pop (9, G). GOAL FOUND! Trace: G←C←B←A←S UCS Path: S→A→B→C→G | Cost: 1+5+2+1 = 9 ✅ OPTIMAL

DFS Procedure

Stack: [S] | Visited: {S} Pop S → expand alphabetically, push in REVERSE order (so A pops first): Push C, then A → Stack: [C, A] | Visited: {S,A,C} Pop A → expand: B(new), C(visited), S(visited) → Push B Stack: [C, B] | Visited: {S,A,C,B} Pop B → expand: C(visited), G(new), A(visited) → Push G Stack: [C, G] | Visited: {S,A,C,B,G} Pop G → GOAL FOUND! Path: G←B←A←S DFS Path: S→A→B→G | Cost: 1+5+5 = 11 // OR S→A→B→C→G depending on expansion order

Final Answer Summary for This Graph

AlgorithmPathCostWhy
BFSS→C→G11Fewest hops (2 edges). Not cheapest.
DFSS→A→B→C→G9Deepest alphabetical path. Coincidentally optimal here.
UCSS→A→B→C→G9Cheapest cumulative cost. Guaranteed optimal.
💡

How to get full marks in graph search questions: (1) Write the data structure at each step. (2) Show which node you pop. (3) Show what you expand and add. (4) Show visited set. (5) State final path and cost. Examiners award 1 mark per correct step shown.

Q3 — A* Search + Heuristic Admissibility

5–7 MARKS · FORMULA BASED

Q3 always involves the same graph with heuristic values (S,A,B,C,D,G with h values). You run A* Tree Search, A* Graph Search, check admissibility, check consistency, analyse combinations. Every step is worth marks.

The Standard Exam Graph (Appears in Fall 2025 Q3 and Assignment Q7)

// Graph edges and heuristic values: S → A: cost 2 h(S) = 6 S → C: cost 2 h(A) = 4 A → B: cost 1 h(B) = 1 A → C: cost 2 h(C) = 3 C → D: cost 2 h(D) = 1 B → G: cost 3 h(G) = 0 D → G: cost 3 // f(n) = g(n) + h(n) ← this is what you ALWAYS compute

A* Tree Search — Full Trace

Open = PQ sorted by f=g+h. No closed set (tree search). Init: push S with g=0, f=0+6=6 Open: [(f=6, g=0, S)] Step 1: Pop S (f=6, g=0). Expand: A(g=2, f=2+4=6), C(g=2, f=2+3=5) Open: [(5,g=2,C), (6,g=2,A)] Step 2: Pop C (f=5, g=2). ← smallest f Expand: D(g=4, f=4+1=5), A(g=4,f=4+4=8 — tree: re-add!) Open: [(5,g=4,D), (6,g=2,A), (8,g=4,A again)] Step 3: Pop D (f=5, g=4). ← tie with f=5, pick D (g=4 > A's g=2? pick by f then g or alpha) Expand: G(g=7, f=7+0=7) Open: [(6,g=2,A), (7,g=7,G via D), (8,g=4,A)] Step 4: Pop A (f=6, g=2). Expand: B(g=3,f=3+1=4), C(g=4,f=4+3=7 — re-add for tree) Open: [(4,g=3,B), (7,...), (7,...), (8,...)] Step 5: Pop B (f=4, g=3). ← smallest f now! Expand: G(g=6, f=6+0=6) Open: [(6,g=6,G via B), (7,g=7,G via D), ...] Step 6: Pop G (f=6, g=6). GOAL! A* Tree Path: S → A → B → G Actual Cost: 2 + 1 + 3 = 6 ✅ OPTIMAL

A* Graph Search — Key Difference

Closed = {} (never re-expand a closed node) Same start. Pop S → close S. Add A(f=6), C(f=5). Pop C → close C. Add D(f=5). Pop D → close D. Add G via D (f=7). Pop A → close A. Add B(f=4). Skip C (already closed). Pop B → close B. Add G via B (f=6). Pop G via B (f=6). GOAL! A* Graph Path: S → A → B → G Actual Cost: 6 ✅ // SAME result here — but NOT always the case! // Check consistency to know if graph search is guaranteed optimal.

Admissibility Check — Every Node

// Find actual shortest cost from each node to G: h*(S) = min(S→A→B→G, S→C→D→G) = min(2+1+3, 2+2+3) = min(6,7) = 6 h*(A) = min(A→B→G, A→C→D→G) = min(1+3, 2+2+3) = min(4,7) = 4 h*(C) = C→D→G = 2+3 = 5 h*(B) = B→G = 3 h*(D) = D→G = 3 h*(G) = 0 // Now check h(n) ≤ h*(n) for each node: h(S)=6 ≤ h*(S)=6 ? YES ✓ h(A)=4 ≤ h*(A)=4 ? YES ✓ h(C)=3 ≤ h*(C)=5 ? YES ✓ h(B)=1 ≤ h*(B)=3 ? YES ✓ h(D)=1 ≤ h*(D)=3 ? YES ✓ h(G)=0 ≤ h*(G)=0 ? YES ✓ → Heuristic IS ADMISSIBLE ✅

Consistency Check — Every Edge

// For each edge n→n': check h(n) ≤ cost(n,n') + h(n') S→A: h(S)=6 ≤ 2+h(A)=2+4=6 ? 6 ≤ 6 YES ✓ S→C: h(S)=6 ≤ 2+h(C)=2+3=5 ? 6 ≤ 5 NO ✗ ← INCONSISTENT! A→B: h(A)=4 ≤ 1+h(B)=1+1=2 ? 4 ≤ 2 NO ✗ ← INCONSISTENT! A→C: h(A)=4 ≤ 2+h(C)=2+3=5 ? 4 ≤ 5 YES ✓ C→D: h(C)=3 ≤ 2+h(D)=2+1=3 ? 3 ≤ 3 YES ✓ B→G: h(B)=1 ≤ 3+h(G)=3+0=3 ? 1 ≤ 3 YES ✓ D→G: h(D)=1 ≤ 3+h(G)=3+0=3 ? 1 ≤ 3 YES ✓ → Heuristic is NOT CONSISTENT (fails on S→C and A→B) → A* Graph Search NOT guaranteed optimal. → A* Tree Search IS optimal (h is admissible).
🏆

Full marks strategy for Q3: Always show your admissibility check table. Always show your consistency check per edge. Explicitly state: "h is admissible because h(n) ≤ h*(n) for all n" OR "h is NOT admissible because h(X) = Y > Z = h*(X)". State the implication for A* optimality.

Q4 — Simulated Annealing Calculation

5–7 MARKS · TABLE METHOD

SA is always a numerical calculation question. Given T formula, RNG formula, neighbours with utility values, show accept/reject for each timestep. Use a table — it guarantees full marks and never confuses examiners.

The 5 Things Always Given in SA Questions

  1. Temperature formula — e.g., T = 100 − 0.2t  |  T = t − 2t² + 67  |  T = 8 − 2t
  2. RNG formula — e.g., N = |sin(90 − 4t)|  |  N = |cos(185 + 2.7t)|  |  Fixed N = 0.5
  3. Current state with utility E(current)
  4. Neighbours with their utilities
  5. Order of neighbour evaluation (alphabetical / left-to-right / given sequence)

The 6-Column Table Method — Use This Always

| t | Neighbour | E(nbr) | T(t) | ΔE=E(nbr)-E(cur) | N=RNG(t) | P=e^(ΔE/T) | Accept? | |---|-----------|--------|-------|------------------|----------|------------|---------| | 0 | C | 13 | 100 | 13-20 = -7 | 0.932 | e^(-7/100)| ? | | 1 | A | 15 | 99.8 | 15-20 = -5 | 0.9976 | e^(-5/99.8)| ? | ...

Q5 — Minimax + Alpha-Beta Pruning

4–6 MARKS · VISUAL METHOD

Every exam has a game tree with Minimax. Your job: (1) compute values bottom-up, (2) identify pruned nodes with alpha-beta. Always show [α, β, v] values. Always state WHY each node is pruned.

The 3-Step Protocol for Any Game Tree

1
Label every node MAX or MIN. Root is always MAX. Levels alternate. Triangles pointing up = MAX. Triangles pointing down = MIN. Squares/circles = leaf (terminal) nodes with given values.
2
Compute all values bottom-up (without pruning first). At MAX nodes: take max of children. At MIN nodes: take min of children. Write the value next to each node. Root value = minimax decision.
3
Re-trace with alpha-beta, tracking (α, β) at each node. α starts at −∞, β at +∞. Update α at MAX nodes (α=max(α,v)). Update β at MIN nodes (β=min(β,v)). When α≥β: prune remaining children, explain why.

Q6 — Mixed Topics (Hill Climbing / CSP / Heuristic Combos)

2–5 MARKS · CONCEPTUAL

This is the "wildcard" question. It tests conceptual understanding of topics from across the syllabus. Based on past papers, it's typically: short conceptual Q about hill climbing variants, OR heuristic combination admissibility, OR a small CSP problem. These require understanding, not calculation.

Most Likely Subtopics

Heuristic Combinations HOT

Given h1,h2,h3 admissible, h4 inadmissible — is max(h1,h4) admissible? Is min(h1,h4)?

Key rules: max with inadmissible → NO. min with inadmissible → YES. Sum with inadmissible → NO. Average with inadmissible → NO.

HC Variants SURE

Sideways moves, random restart, local beam, stochastic HC.

Key answers: Sideways moves → escape plateaux, 14%→94% on 8-queens. Beam k=1 = HC. Beam ≠ k independent restarts.

Temperature & SA Concepts MEDIUM

Role of T, high T vs low T, cooling schedule, SA vs HC comparison.

Key answer: High T = accept bad moves freely. Low T = pure HC. Decreasing T over time allows escaping local optima early, exploiting late.

Minimax Theory MEDIUM

Is minimax always optimal? What if opponent is irrational? Eval functions.

Key answer: Optimal vs rational opponent. NOT optimal vs irrational. NOT optimal with depth-limited + eval function.

Model Answers for All Likely Short Questions

ASSIGNMENT Q4b Beam Search (k) vs Random Restart (k) [2 MARKS]
What is the difference between Randomly Restarting Hill Climbing k times and local beam search with beam size k?
▶ MODEL ANSWER

Random Restart (k times): Runs k completely independent hill climbing searches, each starting from a different random state. No information is shared between runs. After all k runs finish, we pick the best result.

Local Beam Search (k states): Maintains exactly k states simultaneously in memory. At each step, ALL successors of ALL k states are generated, and the k best successors are selected to continue. States actively share information — a state in a promising region generates more offspring, concentrating the search there. This is fundamentally different from k independent searches.

Key distinction: In random restart, the k searches are isolated. In beam search, the k states communicate and influence each other. When k=1, beam search = ordinary hill climbing.

ASSIGNMENT Q4d Role of Temperature T in SA [2 MARKS]
What is the role of temperature T in Simulated Annealing? How do chances of taking bad moves change with T?
▶ MODEL ANSWER

Temperature T controls the probability of accepting bad (worse) moves. The acceptance probability for a bad move is P = e^(ΔE/T) for maximisation.

  • High T (hot): P → e⁰ = 1. Almost all bad moves are accepted. The algorithm explores widely and can escape local optima freely. Acts like a random walk.
  • Low T (cold): P → 0. Bad moves are almost never accepted. The algorithm behaves like pure hill climbing — only improving moves are taken.
  • Decreasing T over time (cooling schedule): Starts by exploring broadly, gradually converges to exploiting the best region found. If T decreases slowly enough, SA is theoretically guaranteed to find the global optimum.

For combinations: algebraically simplify first. If inadmissible h4 appears with positive coefficient in a sum/max → likely NOT admissible. If h4 only appears in a min with admissible functions → admissible. Always show the algebraic simplification step.

PREDICTED Q4 Simulated Annealing — Show t=0 to t=N calculations [5–7 MARKS]

Likely scenario: Given current node with utility, 4–6 neighbours with utilities. T = some formula in t. N = some |sin| or |cos| formula. Neighbours checked in alphabetical order. Show accept/reject for each timestep until first accepted move (part a). Choose your own neighbour for t=N and show calculation (part b).

✅

Use the 6-column table format: t | Neighbour | E | T | ΔE | N | P | Decision. Compute T and N first (they're mechanical). The only real work is computing e^(ΔE/T). If ΔE > 0, always accept immediately without computing P.

PREDICTED Q5 Minimax + Alpha-Beta Pruning with [α,β,v] values [4–6 MARKS]

Likely scenario: A 3–5 level game tree (MAX/MIN alternating). ~8–15 leaf nodes with given values. Apply Minimax and Alpha-Beta. Show [α,β,v] at each internal node. State which nodes are pruned AND WHY (state the condition). Part (b): "Does minimax always give optimal?" OR "What are the limitations?"

✅

The exam may say "children visited right-to-left" — this changes which nodes get pruned but NOT the root value. For pruning explanation: always say "α(X) ≥ β(Y) at node Z, therefore remaining children of Z are pruned because [MAX/MIN] already has a better option."

Magic Keywords — Exact Phrases That Get Full Marks

Examiners look for specific terminology. Using the right word signals understanding. Using vague language loses marks even if you know the concept. Here are the exact phrases to use.

For Search Questions

Instead of saying…Say exactly this…
"BFS finds the shortest path""BFS finds the path with the minimum number of actions (shallowest solution), but is only optimal when all edge costs are equal."
"UCS is like BFS but for costs""UCS expands nodes in order of increasing cumulative path cost g(n), and is both complete and optimal when all edge costs are positive."
"DFS might not find the best path""DFS is neither complete (may loop in infinite graphs) nor optimal — it returns the first path found, not the cheapest."
"A* is better than UCS""A* is more efficient than UCS because it uses a heuristic h(n) to prioritise nodes closer to the goal, reducing the number of expansions while maintaining optimality."

For Heuristic Questions

ConceptExact phrase for full marks
Admissibility"The heuristic h is admissible because h(n) ≤ h*(n) for all nodes n, where h*(n) is the true optimal cost from n to the goal. It never overestimates."
Not admissible"The heuristic h is NOT admissible because h(C) = 20 > 9 = h*(C). Node C's actual shortest path cost to goal is 9, but h overestimates at 20."
Consistency"The heuristic is NOT consistent because h(S) = 6 > c(S→C) + h(C) = 2+3 = 5, violating the triangle inequality h(n) ≤ c(n,n') + h(n')."
A* Graph not optimal"A* Graph Search is NOT guaranteed optimal here because h is inconsistent. An inconsistent heuristic may cause the graph search to close a node via a suboptimal path, preventing rediscovery of the optimal path."

For SA Questions

SituationWhat to write
Good move (ΔE > 0, maximisation)"ΔE = +X > 0, so this is a better move. In SA, better moves are always accepted unconditionally. Move to [neighbour]."
Bad move, accepted"ΔE = −X < 0 (worse move). P = e^(ΔE/T) = e^(−X/T) = Y. Since RNG = Z < P = Y, the bad move is ACCEPTED."
Bad move, rejected"ΔE = −X < 0. P = e^(ΔE/T) = Y. Since RNG = Z ≥ P = Y, the bad move is REJECTED. Stay at current state."

For Minimax + Alpha-Beta

SituationWhat to write
Pruning at MIN node"At MIN node [X], β = min(β, v) = Y. Since α(Z) ≥ β(Y), remaining children of [X] are pruned. MAX already has a guaranteed path worth Z, so it will never choose this MIN node which can give at most Y ≤ Z."
Pruning at MAX node"At MAX node [X], α = max(α, v) = Y. Since α(Y) ≥ β(Z), remaining children of [X] are pruned. MIN already has a path worth Z ≤ Y, so it will never let MAX reach this node."
Minimax not always optimal"Minimax is optimal only against a perfectly rational opponent. Against an irrational opponent, or when using depth-limited search with an evaluation function, the guarantee of optimality is lost."

The 5 Golden Rules of Getting Full Marks

1
Show every single step of your working. Examiners cannot give marks for steps they cannot see. Even if you know the answer, write how you got there. Each visible step = potential mark.
2
State conclusions explicitly. Don't assume the examiner will infer. After checking admissibility: write "Therefore, the heuristic IS/IS NOT admissible." After finding a path: write "UCS returns path S→A→B→C→G with total cost 9."
3
Never leave a question blank. Even if you don't know the exact answer, write the framework. For PEAS: write any reasonable sensors and actuators. For SA: write the formula even if you get the numbers slightly wrong. Partial credit exists.
4
Answer in the order that suits you, not the paper. Do your strongest question first (probably Q1 + Q2). Leave your weakest for last. Never get stuck on Q2 for 30 minutes when Q1 is a guaranteed 5 marks.
5
Use exact terminology from the course. "Admissible," "consistent," "complete," "optimal," "alpha-beta cutoff," "priority queue," "utility," "minimax value" — these words tell the examiner you understand the concept, not just the calculation.
