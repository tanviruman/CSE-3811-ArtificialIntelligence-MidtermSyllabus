What is Artificial Intelligence?

Definition: AI attempts to build intelligent entities — programs and machines that can creatively perceive, reason, and solve problems. Term coined by John McCarthy at Dartmouth College, 1956.

This course focus: Building rational agents — agents that always choose the action that maximises their expected outcome. Not "act human," but act correctly.

AI vs ML vs Deep Learning

TermWhat it meansExample
Artificial Intelligence (AI)Build intelligent programs that creatively solve problemsChess engine, self-driving car
Machine Learning (ML)Systems that automatically learn and improve from data — without being explicitly reprogrammedNetflix recommendations, spam filter
Deep Learning (DL)ML using deep neural networks to learn from raw dataImage recognition, ChatGPT

Memory: AI ⊃ ML ⊃ DL. Every DL system is ML, every ML system is AI. But not every AI system uses DL.

The Turing Test

Core idea: A computer passes the Turing Test if a human interrogator — asking written questions — cannot tell whether responses come from a human or a machine. Proposed by Alan Turing (1950).

From your handwritten notes: "How much does an AI agent's answer differ from a human?" — Example: CAPTCHA (Completely Automated Public Turing test to tell Computers and Humans Apart) is based on this idea.

What Capabilities Does Passing the Turing Test Require?

  • Natural Language Processing (NLP) — Understand and generate human language
  • Knowledge Representation — Store and recall world knowledge
  • Automated Reasoning — Use stored knowledge to draw conclusions
  • Machine Learning — Adapt to new situations
  • Computer Vision — Perceive visual input (for Total Turing Test)
  • Robotics — Physically interact with the world (for Total Turing Test)

Why AI researchers mostly ignore the Turing Test: It is not reproducible, not constructive, and cannot be mathematically analyzed. It is more important to study the underlying principles of intelligence than to mimic human behavior.

The 4 Views of AI — Most Exam Questions Come From Here!

AI can be defined in 4 ways based on two axes: Thinking vs Acting and Human-like vs Rational.

🧠 Thinking Humanly

AI that models and replicates how the human brain works. Studies cognitive science. Simulation of human thought processes including errors.

Example: AI that makes memory errors like humans do.

⚖️ Thinking Rationally

Uses formal logic / "laws of thought" to derive correct conclusions from premises. Inference-based reasoning.

Example: AI that solves a math theorem logically.

🎭 Acting Humanly

Produces behavior indistinguishable from a human. Passes the Turing Test. Mimics human behavior patterns.

Examples: Siri telling jokes, ChatGPT writing friendly tone, customer-service bot apologising politely.

🎯 Acting Rationally ⭐ THIS COURSE

Takes the best possible action to achieve a goal. Maximises expected utility. Does NOT need to mimic humans at all.

Examples: Self-driving car, Google Maps, Netflix recommendation.

Basic Design of an Agent

An agent is anything that can be viewed as perceiving its environment through sensors and acting upon that environment through actuators. The agent processes percepts and takes actions.

From your notes: Actuator = module that shows output / takes action. Sensor = how input is taken.

Types of Agents

Agent TypeSensors (Input)Actuators (Output)
Human AgentEyes, ears, skin, noseHands, legs, vocal tract, face
Robotic AgentCameras, IR sensors, GPS, microphoneMotors, wheels, robotic arms, speakers
Software AgentKeystrokes, file contents, network packetsDisplay on screen, write files, send network packets
⚠️

Common mix-up in your notes: You wrote "Actuator - eyes, ears / Sensor - hands, legs" — this is REVERSED! Sensors = input (eyes, ears) and Actuators = output (hands, legs, vocal tract).

The Concept of Rationality

Rational agent = one that does the right thing — selects actions that maximise its expected utility (performance measure), given what it knows. Rationality is about results, not the process of thinking.

PEAS Framework — How to Describe Any Agent

PEAS = the standard way to formally describe an AI agent. You will be asked to write PEAS descriptions in every exam. Master this!

P — PERFORMANCE MEASURE

How do we judge if the agent is doing a good job? What score does it get? What is it optimising?
Ask: "What should the agent achieve?"

E — ENVIRONMENT

Where does the agent live and operate? What is around it? What world does it interact with?
Ask: "Where does the agent act?"

A — ACTUATORS

What can the agent DO? Its output mechanisms — how does it take action on the world?
Ask: "What tools does it use to act?"

S — SENSORS

What can the agent PERCEIVE? Its input mechanisms — how does it get information from the environment?
Ask: "How does it see/hear/feel the world?"

Memory trick: "Please Explain Agent's Structure" = Performance, Environment, Actuators, Sensors

7 Environment Types — Exam MUST KNOW!

Each environment type defines what kind of challenge the agent faces. You must be able to classify any given scenario using these 7 properties. From your handwritten notes!

1. Fully Observable vs Partially Observable

✅ Fully Observable

Agent's sensors give access to the complete state of the environment at all times. Nothing is hidden.

Examples: Chess, crossword puzzle, Google Maps with full traffic data.

Your notes example: Google self-driving car.

⚠️ Partially Observable

Agent can only perceive a partial area or subset of the environment. Some information is hidden.

Examples: Poker (can't see other players' cards), driving in fog, medical diagnosis.

Your notes: "Partial Area — Not everything."

2. Single Agent vs Multi-Agent

👤 Single Agent

Only one agent operates in the environment. No other decision-making entities.

Example: BayMax from Big Hero 6 — acts alone without help from others. Crossword puzzle solver.

👥 Multi-Agent

Multiple agents interact. Can be cooperative (helping each other) or competitive (opposing goals).

Examples: Chess (competitive), soccer robots (cooperative), taxi fleet, stock trading.

3. Deterministic vs Stochastic

🎯 Deterministic

Next state is completely determined by the current state and agent's action. No randomness.

Example: Apple Words game (fill in blank spaces). Chess moves are deterministic.

🎲 Stochastic

Outcomes are uncertain / random. Same action may produce different results.

Examples: Soccer board penalty, driving (other cars unpredictable), dice games.

Your notes: "Doesn't tell exact result."

4. Episodic vs Sequential

📸 Episodic

Each episode is independent. Current action does NOT affect future episodes. Agent just perceive + act.

Examples: Soccer match scoring (each match separate), spam filter, image classification.

Your notes: "End one round."

📺 Sequential

Current decisions affect future decisions. Actions have long-term consequences. History matters.

Examples: Chess (each move affects all future moves), Game of Thrones character decisions, medical treatment plans.

5. Static vs Dynamic

🧊 Static

Environment does NOT change while the agent is thinking/deliberating. World waits for agent.

Examples: Crossword puzzle, chess (technically semi-static with clock).

Your notes: "Still."

🌊 Dynamic

Environment changes while agent deliberates. Agent must respond to continuous changes.

Examples: Driving, stock trading, soccer (world changes every millisecond).

Your notes: "Not still."

6. Discrete vs Continuous

🔢 Discrete

Finite number of distinct, clearly defined states/actions. Countable choices.

Examples: Chess (64 squares, finite pieces), card games.

Your notes: "Option."

📈 Continuous

States and/or actions are defined over a continuous range. Infinite possible values.

Examples: Robot arm angle (27.3°), temperature (27°C — your notes), car steering angle.

Your notes example: 27°C = continuous.

7. Known vs Unknown

📚 Known

The agent knows the rules / physics / outcomes of its actions in advance.

Example: Chess — you know exactly what each move does. Card game with known rules.

❓ Unknown

Agent must learn the effects of actions through experience. Rules not pre-known.

Example: Exploring a new physical environment, playing an unknown card game for the first time.

Problem Formulation — The 5 Components

Problem Formulation = a strategy/description defined before solving a problem using search. It's how we tell a computer what the problem is so it can search for a solution.

From your notes: "Problem strategy before solving a problem." It defines Start State, Actions, Transition, Goal Test, Path Cost.

ComponentWhat It IsExample: 2D Grid Navigation
State RepresentationHow do we describe "where we are" in the problem?Grid coordinates (i, j)
Total # of StatesHow many possible configurations exist?All valid (i,j) grid positions
ActionsWhat moves can be made from any state?Up and Right (from your notes: "up and right")
Successor / Transition FunctionGiven current state + action → what is the next state? (i,j) →up (i,j+1) | (i,j) →right (i+1,j)Moving up increases j by 1; moving right increases i by 1
Start StateWhere do we begin?(0,0) — bottom-left
Goal TestHow do we know we've succeeded?(i,j) == (2,2)? — reaching top-right
Path CostHow expensive is the solution? What are we minimising?# of moves (number of steps taken)

⏰ Time Machine Problem (From Midterm)

Okabe is in year 2025, wants to reach year 2000 in at most 4 jumps. Each jump can go to any year.

State(current_year, jumps_used) — tracks both position in time AND how many jumps used
Start State(2025, 0)
ActionsJump to any year y (any integer, past or future)
Successor Function(year, j) → (y, j+1) for any year y, as long as j+1 ≤ 4
Goal Testcurrent_year == 2000 AND jumps_used ≤ 4
Path CostNumber of jumps used (minimise this)

Example solution: 2025 → 2001 → 1997 → 2000 (3 jumps). Or simply: 2025 → 2000 (1 jump — optimal!)

🚁 Delivery Drone Problem

State(drone_location, set_of_undelivered_packages)
Start State(depot, all_packages_undelivered)
ActionsFly to adjacent city node; Deliver package at current location
Successor FunctionMove(v) → location becomes v; Deliver(p) → p removed from undelivered set
Goal TestAll packages delivered (undelivered set is empty)
Path CostTotal distance + fuel consumed
