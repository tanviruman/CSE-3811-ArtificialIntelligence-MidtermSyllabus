What is a Search Problem?

A search problem consists of a state space, a successor function (with actions and costs), a start state, and a goal test. A solution is a sequence of actions (a plan) which transforms the start state to a goal state.

Why Do We Need Search?

Many AI problems can't be solved directly with a formula. Instead, the agent must explore possible sequences of actions to find one that achieves the goal. Search is the backbone of classical AI problem solving.

Pacman Example (From Your Notes — Class 3)

Pacman navigating a maze is a classic search problem:

ComponentPacman Problem: Find PathPacman Problem: Eat All Dots
States(x, y) location on grid{(x,y), dot booleans} — position + which dots remain
ActionsNorth, South, East, WestNorth, South, East, West
SuccessorUpdate location onlyUpdate location + possibly remove a dot
Goal TestIs (x,y) = END?Are all dots eaten (all booleans false)?
💡

Key insight: The "world state" includes every detail of the environment. A "search state" keeps only what's needed for planning — we abstract away irrelevant details.

🌳

State Space Graph vs Search Tree

📊 State Space Graph

A mathematical map of the entire problem. Each state appears exactly ONCE. Nodes = world configurations. Edges = possible actions (transitions).

Cannot always build this in memory — too large! But it's a useful conceptual tool.

🌲 Search Tree

A "what if" tree of plans. The start state is the root. Children correspond to successors. A state can appear MULTIPLE times (via different paths). Nodes represent plans, not just states.

From your notes: "Infinite graph" — the tree can be infinite even if the graph is finite!

Why does this matter? When doing graph search vs tree search: Graph Search keeps a "closed/visited" set — never expands the same state twice. Tree Search may revisit states (no closed set) — can loop infinitely but simpler.

From Your Handwritten Notes (Class 3)

// Example: Graph with 4 nodes: S, a, b, G // S connects to a and b; both a and b connect to G State Space Graph: Search Tree (infinite!): S S / \ / \ a b a b \ / /\ /\ G b G a G ...repeated forever
🧠

Tree Search vs Graph Search: Tree search allows repeated states (simple but may loop). Graph search uses a visited/closed set to avoid repeating states (more efficient, avoids loops).

4 Search Algorithm Properties

Every search algorithm is evaluated on these 4 criteria. Know them by heart — they appear in every exam!

Variables: b = branching factor (average children per node), m = maximum depth of tree, s = depth of shallowest solution, L = depth limit for DLS.

PropertyQuestion It AnswersDetails
Completeness"Will it ALWAYS find a solution if one exists?"If yes → guaranteed to find goal. If no → might miss it or loop forever.
Optimality"Will it find the BEST (cheapest) solution?"If yes → returns minimum cost path. If no → might find any path, not the best.
Time Complexity"How many nodes are explored?"Big-O notation. Smaller = faster algorithm.
Space Complexity"How much memory is used?"How many nodes stored simultaneously. Smaller = less memory needed.
⚠️

Optimality vs Completeness: An algorithm can be complete but not optimal (finds A solution, but not the best). Optimality requires BOTH finding a solution AND it being cheapest.

BFS — Breadth-First Search

Core Idea: Explore the search tree level by level. First check all nodes at depth 1, then all at depth 2, etc. Like spreading water — it floods outward uniformly.

Data Structure: Queue (FIFO) — First In, First Out. Newly discovered nodes go to the BACK of the queue.

How BFS Works Step by Step

1

Add the start node to the Queue with cost 0

2

Take the FRONT node from the Queue (oldest node)

3

Check: Is it the goal? If YES → done! If NO → continue

4

Add all unvisited children/neighbours to the BACK of the Queue

5

Mark current node as visited

6

Repeat from step 2 until queue is empty or goal found

BFS Properties

Completeness

✓ YES — Will always find a solution if one exists (assuming finite branching factor b).

Optimality

CONDITIONAL — Optimal ONLY when all step costs are equal (e.g., all = 1). If costs differ, use UCS instead!

Time Complexity

O(bs)

Where s = depth of shallowest solution. Explores all nodes up to depth s.

Space Complexity

O(bs)

🔴 HIGH! Must store entire frontier. The biggest weakness of BFS.

BFS Visualisation — 3-Level Tree (b=2)

// BFS explores level by level: Level 0: [A] ← expand A Level 1: [B, C] ← expand B, then C Level 2: [D, E, F, G] ← expand D, E, F, G ↑ Goal found here! (cost = 2 steps) Queue at each step: Init: [A] → Pop A: add B,C → Queue: [B, C] → Pop B: add D,E → Queue: [C, D, E] → Pop C: add F,G → Queue: [D, E, F, G] → Pop D: not goal → ... → Pop E: not goal → ... → Pop F: GOAL! → done!
💡

BFS use case: When you need the solution with the fewest steps (or when all costs are equal). Great for: word ladders, shortest path in unweighted graphs, maze with equal step costs.

BFS vs DFS — When to Prefer Which?

ScenarioBFS Better?DFS Better?
Finding shortest path (equal cost)✓ Yes✗ No
Shallow and wide search space✓ Yes✗ No
Memory is very limited✗ No✓ Yes
Deep and narrow tree✗ No (too slow)✓ Yes
Solving a maze✗ Overkill✓ Yes — natural fit

DFS — Depth-First Search

Core Idea: Explore as deep as possible down one path before backtracking. Like exploring a cave — go as far as you can, then come back and try another tunnel.

Data Structure: Stack (LIFO) — Last In, First Out. Newly discovered nodes go to the TOP of the stack.

How DFS Works Step by Step

1

Push start node onto the Stack

2

Pop the TOP node from the Stack (most recently added)

3

Check: Is it the goal? If YES → done! Return path.

4

Push all unvisited children to the TOP of the Stack

5

Repeat from step 2 (goes DEEP because newest is always popped)

DFS Properties

Completeness

✗ NO (infinite spaces) — Can get stuck going down an infinite path forever! COMPLETE on finite trees

Your notes: "DFS is complete if there is no cycle."

Optimality

✗ NO — May find a solution that is NOT the cheapest. Finds the leftmost deep solution, not the best.

Your notes: "No optimal."

Time Complexity

O(bm)

Where m = maximum depth. In worst case, explores entire tree.

Space Complexity

O(b × m)

🟢 LOW! Only stores current path + siblings. This is DFS's big advantage.

Your notes: "Space Complexity = O(b × m)"

DFS Visualisation

// Tree: S→{A,B}, A→{C,D}, B→{E,F}, goal=F Stack at each step: Init: [S] → Pop S: push A, B (B on top) → Stack: [A, B] (B on top) actually depends on order... let's say push right-then-left → Pop B: push E, F (F on top) → Stack: [A, E, F] → Pop F: GOAL! Done. // DFS went: S → B → F (found goal quickly here) // But could explore entire subtree before finding goal! // Space: only stores [S, B, F] — just the current path // Compare BFS: would store entire level-by-level frontier
⚠️

DFS's fatal flaw: In infinite or cyclic graphs, DFS may loop forever. Always use Graph Search (visited set) with DFS to avoid revisiting states.

DLS — Depth-Limited Search

Core Idea: DFS but with a hard depth limit L. Don't expand nodes deeper than L. This prevents the infinite-loop problem of DFS.

From your notes: "DFS but up to certain depth." "Complete — if goal within range."

How DLS is Different from DFS

// DFS: goes as deep as possible (no limit) // DLS: same as DFS BUT stops at depth L DLS with L = 3: Level 0: S ← expand Level 1: A, B ← expand Level 2: C, D, E, F ← expand Level 3: G, H, ... ← STOP! Don't go deeper than 3 // If goal is at depth 4 → DLS with L=3 FAILS to find it // If goal is at depth 2 → DLS with L=3 FINDS it ✓

DLS Properties

PropertyResultExplanation
CompletenessIf L ≥ sComplete only if depth limit L is at least as deep as the shallowest solution s. If L < s, it fails.
Optimality✗ NoDoes not guarantee finding the cheapest solution.
TimeO(bL)Explores all nodes up to depth L.
SpaceO(b × L)Low memory — just stores current path + siblings up to depth L.

DLS in Exam Questions

In your Summer 2025 midterm: DLS with limit=5 on a warehouse grid. Start=(0,0), Goal=(2,1). The question asks what path DLS returns with limit 5. Since the optimal path has length 3 ≤ 5, DLS will find it (follows DFS order with Up>Down>Right>Left priority).

🔁

IDS — Iterative Deepening Search

Core Idea: Run DLS with limit 0, then limit 1, then limit 2, then 3... keeping going until the solution is found. Gets DFS's memory advantage + BFS's completeness and optimality. Best of both worlds!

From your notes: "IDS = CDFS at level = 1, 2, 3, 4 — ..." (Consecutive DFS at increasing levels)

Why IDS Seems Wasteful But Isn't

// IDS runs DLS repeatedly, re-exploring previous levels // "Isn't that wastefully redundant?" — NO! Here's why: IDS iterations (b=2, solution at depth s=3): Depth 0: explore 1 node Depth 1: explore 1+2 = 3 nodes Depth 2: explore 1+2+4 = 7 nodes Depth 3: explore 1+2+4+8 = 15 nodes ← goal found here! Total nodes explored = 1 + 3 + 7 + 15 = 26 // BFS would explore: 1 + 2 + 4 + 8 = 15 nodes (just once) // IDS: 26 nodes — only 73% more work but MUCH less memory! // Key insight: most work happens in the DEEPEST level searched // So re-exploring shallower levels is negligible overhead!

IDS Properties

Completeness

✓ YES — Always finds a solution if one exists. Like BFS!

Optimality

If equal costs — Finds shallowest solution first (optimal when all steps cost 1).

Time Complexity

O(bs)

Same as BFS in time — just slightly more constant factor due to re-exploration.

Space Complexity

O(b × s)

🟢 LOW! Like DFS — only stores current path. This is the big win!

⭐

IDS is the best uninformed search algorithm in practice! Use it when you don't know the solution depth and want both completeness and low memory.

⚡

Complete Comparison Table

AlgorithmData StructureComplete?Optimal?TimeSpaceBest Use Case
BFS Queue (FIFO) ✓ Yes If cost=1 O(bs) O(bs) HIGH Shortest path (equal costs)
DFS Stack (LIFO) ✗ No* ✗ No O(bm) O(bm) LOW Memory limited, finite spaces
DLS Stack + depth counter If L≥s ✗ No O(bL) O(bL) LOW Known max depth (e.g., limit=5)
IDS Stack (like DFS) ✓ Yes If cost=1 O(bs) O(bs) LOW ⭐ Unknown depth, need completeness

*DFS is complete on finite trees without cycles. With a visited set (graph search), DFS becomes complete on finite graphs too.

🧠

Memory tip: BFS = horizontal (wide, lots of memory). DFS = vertical (deep, low memory). IDS = DFS trying every depth limit = low memory + complete!

Solved Exam Problems

Problem 1: IDS Node Calculation (b=4, m=10, s=3)

Search tree: branching factor b=4, max depth m=10, shallowest solution depth s=3. Root at level 0, algorithms work left to right. Which strategy expands fewest nodes? Expanding any node has unit cost.

▶ SOLUTION

Case a: Shallowest solution at LEFTMOST node (depth s=3)

If goal is leftmost at depth 3, DFS going left-to-right finds it by going straight down the leftmost branch — very quickly (about 3×4 ≈ 13 nodes). DFS wins here!

Case b: Shallowest solution at RIGHTMOST node (depth s=3)

DFS will explore ALL branches from left before reaching rightmost. BFS or IDS performs better here.

IDS Node Count (worst case, b=4, s=3):

Iteration 0: 1 node
Iteration 1: 1 + 4 = 5 nodes
Iteration 2: 1 + 4 + 16 = 21 nodes
Iteration 3: 1 + 4 + 16 + 64 = 85 nodes

Total IDS = 1 + 5 + 21 + 85 = 112 nodes

BFS node count: 1 + 4 + 16 + 64 = 85 nodes (but requires O(b³) = 64 nodes in memory at once!)

Key insight: IDS uses only O(bs) = 12 memory slots. BFS needs O(b³) = 64 memory slots. IDS trades a small time overhead (112 vs 85 nodes) for huge memory savings. That's why IDS is preferred!

Case: infinite depth (m = ∞):

BFS and IDS still work (they find shallowest solution). DFS fails — it may go infinitely deep and never return.

Problem 2: DFS/BFS on Password Recovery (From Spring 2025 Midterm)

Strings of X,Y,Z. DFS/BFS/UCS search for passwords. Alphabetical tie-breaking. Correct passwords: XXXZZZ, XYYZZ, YXYXY, YZXYXZY, ZYXZ, ZYXZY.

▶ SOLUTION

DFS (goes deepest, alphabetical = X before Y before Z):

DFS explores X→XX→XXX→XXXX→...→XXXXXXXXXX (deepest X path), then backtracks. Eventually reaches XXXZZZ (correct!). DFS returns XXXZZZ.

BFS (explores level by level, alphabetical tie-breaking):

BFS checks all length-1 strings (X,Y,Z), then all length-2, then length-3, then length-4. The only correct password of length 4 is ZYXZ. BFS returns ZYXZ.

✅ BFS finds shallowest (shortest) correct password. ✅ DFS finds the leftmost deep correct password (along the all-X path).
